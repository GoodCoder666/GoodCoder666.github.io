C - Cards Query Problem

题目大意

有 NNN 个盒子，编号 1∼N1\sim N1∼N，初始均为空。依次处理 QQQ 次询问：

  • 1 i j：将数字 iii 写在一张空卡牌上，放入盒子 jjj。
  • 2 i：按升序输出盒子 iii 中的所有卡牌（允许重复）。
  • 3 i：按升序输出包含卡牌 iii 的所有盒子的编号。若一个盒子里有多张卡牌 iii，则这个盒子的编号仅输出一次。

1≤N,Q≤2×1051\le N,Q\le 2\times 10^51≤N,Q≤2×105

对于查询中所有卡牌上的数字 xxx，均有 1≤x≤2×1051\le x\le 2\times 10^51≤x≤2×105。
对于查询中所有的盒子编号 yyy，均有 1≤y≤N1\le y\le N1≤y≤N。

题目保证输出不超过 2×1052\times 10^52×105 个整数。

分析

我们分别考虑两种输出操作的做法。

  • 2 i：很容易想到，既然要按升序输出，并且允许重复，我们可以使用 NNN 个 multiset 来依次存储每个盒子中的卡牌，处理操作 111 时更新。
  • 3 i：首先不能从 NNN 个盒子中依次查找，这样明显会 TLE。正确的做法是，使用 2×1052\times 10^52×105 个 set（注意不能重复，所以不用 multiset）分别存储每张卡牌所在的箱子编号，处理操作 111 时更新。

此外，本题也可以使用 priority_queue、map，甚至直接输出时排序并去重，不过使用 set 的方式是最简单、代码量最少的。几种方法的总时间复杂度都是 O(Qlog⁡N)\mathcal O(Q\log N)O(QlogN)。

代码

#include <cstdio>
#include <set>
#define maxn 200005
using namespace std;

multiset<int> box[maxn];
set<int> has[maxn];

int main()
{
	int n, q;
	scanf("%d%d", &n, &q);
	while(q--)
	{
		int op, i;
		scanf("%d%d", &op, &i);
		if(op == 1)
		{
			int j;
			scanf("%d", &j);
			box[j].insert(i);
			has[i].insert(j);
		}
		else if(op == 2)
		{
			for(int x: box[i])
				printf("%d ", x);
			putchar('\n');
		}
		else if(op == 3)
		{
			for(int x: has[i])
				printf("%d ", x);
			putchar('\n');
		}
	}
	return 0;
}

D - Writing a Numeral

题目大意

我们有一个字符串 SSS。初始时，S=S=S= 1。

处理如下 QQQ 次询问：

  • 1 x：将数字 xxx 追加至 SSS 的最后面。保证 x∈{1,2,3,4,5,6,7,8,9}x \in \{1,2,3,4,5,6,7,8,9\}x∈{1,2,3,4,5,6,7,8,9}。
  • 2：删除 SSS 的第一个字符。保证此时 ∣S∣>1|S|>1∣S∣>1。
  • 3：输出 SSS 在十进制中对应的数字，对 998244353998244353998244353 取模。

1≤Q≤6×1051\le Q\le 6\times 10^51≤Q≤6×105

分析

首先，我们必须使用一个 queue 或 deque 来存储字符串 SSS。然后，为了在 O(1)\mathcal O(1)O(1) 的时间内处理第三种操作，我们必须维护 S mod (P=998244353)S \bmod (P=998244353)Smod(P=998244353) 的值，记为 AAA。下面考虑前两种操作对 AAA 的影响：

  • 1 x：只需在十进制中腾出一位 000 再加上 xxx 即可，可表示为 A←(10A+x) mod PA \leftarrow (10A+x)\bmod PA←(10A+x)modP。
  • 2：先从队列中取出 SSS 的第一位，记为 xxx。我们需要从 AAA 中减掉最高位乘上其在十进制中的权值，即 A←(A−10∣S∣x) mod PA \leftarrow (A-10^{|S|}x)\bmod PA←(A−10∣S∣x)modP（此时 ∣S∣|S|∣S∣ 表示队列取出前一位后的长度，等同于取出前的 ∣S∣−1|S|-1∣S∣−1）

对于 10n10^n10n 的计算，我们可以用一个变量实时维护 10∣S∣ mod P10^{|S|}\bmod P10∣S∣modP 的值，也可以预处理出所有 10n mod P10^n \bmod P10nmodP，或者直接使用快速幂。

总时间复杂度为 O(Qlog⁡Q)\mathcal O(Q\log Q)O(QlogQ)（快速幂）或 O(Q)\mathcal O(Q)O(Q)（预处理）。

E - Unfair Sugoroku

题目大意

Takahashi 和 Aoki 将玩一个游戏。游戏规则如下：

  • 游戏棋盘有 NNN 个点（编号 1∼N1\sim N1∼N），两玩家轮流投骰子并前进。
  • Takahashi 初始在点 AAA，Aoki 初始在点 BBB。
  • Takahashi 的骰子等概率出现 1,2,…,P1,2,\dots,P1,2,…,P，Aoki 的骰子等概率出现 1,2,…,Q1,2,\dots,Q1,2,…,Q。
  • 当一个玩家当前在点 xxx 且骰子出现 iii 时，他移动到点 min⁡(x+i,N)\min(x+i,N)min(x+i,N)。
  • 先到达点 NNN 的玩家胜利。

假定 Takahashi 先行，求他赢的概率，对 998244353998244353998244353 取模。

2≤N≤1002\le N\le 1002≤N≤100
1≤A,B≤N1\le A,B\le N1≤A,B≤N
1≤P,Q≤101\le P,Q\le 101≤P,Q≤10

分析

自己的赛时解法太复杂了，这里介绍官方题解的做法。

考虑概率 DP（下面用 Ta 表示 Takahashi，Ao 表示 Aoki）：

  • 令 fi,jf_{i,j}fi,j​ 表示 Ta 在点 iii，Ao 在点 jjj，下一轮 Ta 移动时 Ta 获胜的概率。
  • 令 gi,jg_{i,j}gi,j​ 表示 Ta 在点 iii，Ao 在点 jjj，下一轮 Ao 移动时 Ta 获胜的概率。

首先考虑初始状态。根据游戏规则，对于任意 1≤i<n1\le i<n1≤i<n，fn,i=gn,i=1,fi,n=gi,n=0f_{n,i}=g_{n,i}=1,f_{i,n}=g_{i,n}=0fn,i​=gn,i​=1,fi,n​=gi,n​=0。

  • 对于 Ao 当前走的每种可能的步数 k=1,2,…,Qk=1,2,\dots,Qk=1,2,…,Q，有 gi,j:=gi,j+1Qfi,min⁡(j+k,N)g_{i,j}:=g_{i,j}+\frac1Qf_{i,\min(j+k,N)}gi,j​:=gi,j​+Q1​fi,min(j+k,N)​。

最终输出结果即为 fp,qf_{p,q}fp,q​。总时间复杂度为 O(N2(P+Q))\mathcal O(N^2(P+Q))O(N2(P+Q))。使用前缀和可以优化到 O(N2)\mathcal O(N^2)O(N2)，有兴趣的可以自己尝试，这里不详细解释了。

#include <cstdio>
#include <algorithm>
#define MOD 998244353
#define maxn 105
using namespace std;

using LL = long long;
inline LL inv(LL x) // x ^ (MOD - 2) % MOD
{
	int y = MOD - 2;
	LL res = 1LL;
	while(y)
	{
		if(y & 1) (res *= x) %= MOD;
		(x *= x) %= MOD, y >>= 1;
	}
	return res;
}

inline void add(int& x, int y)
{
	if((x += y) >= MOD)
		x -= MOD;
}

int f[maxn][maxn], g[maxn][maxn];

int main()
{
	int n, a, b, p, q;
	scanf("%d%d%d%d%d", &n, &a, &b, &p, &q);
	for(int i=0; i<n; i++)
		f[n][i] = g[n][i] = 1, f[i][n] = g[i][n] = 0;
	LL prob_p = inv(p), prob_q = inv(q);
	for(int i=n-1; i>=a; i--)
		for(int j=n-1; j>=b; j--)
		{
			for(int k=1; k<=p; k++)
				add(f[i][j], g[min(i + k, n)][j]);
			f[i][j] = f[i][j] * prob_p % MOD;
			for(int k=1; k<=q; k++)
				add(g[i][j], f[i][min(j + k, n)]);
			g[i][j] = g[i][j] * prob_q % MOD;
		}
	printf("%d\n", f[a][b]);
	return 0;
}

F - Rook Score

题目大意

有一个 109×10910^9\times 10^9109×109 的网格。令 (i,j)(i,j)(i,j) 表示第 iii 行 jjj 列的格子（1≤i,j≤1091\le i,j\le 10^91≤i,j≤109）。

对于 i=1,2,…,Ni=1,2,\dots,Ni=1,2,…,N，整数 xix_ixi​ 被写在 (ri,ci)(r_i,c_i)(ri​,ci​) 上。在剩余的 1018−N10^{18}-N1018−N 个格子里只有数字 000。

你可以选择一个格子 (R,C)(R,C)(R,C) 并计算与其同行或同列的 2×109−12\times 10^9-12×109−1 个整数之和 SSS。

求最大可能的 SSS。

1≤N≤2×1051\le N\le 2\times 10^51≤N≤2×105
1≤ri,ci,xi≤1091\le r_i,c_i,x_i\le 10^91≤ri​,ci​,xi​≤109
(ri,ci)≠(rj,cj)      (i≠j)(r_i,c_i)\ne (r_j,c_j)~~~~~~(i\ne j)(ri​,ci​)​=(rj​,cj​)      (i​=j)

分析

我们令 f(R,C)f(R,C)f(R,C) 表示对于 (R,C)(R,C)(R,C) 的 SSS，令 rsR\mathrm{rs}_RrsR​ 表示第 RRR 行的整数之和，csC\mathrm{cs}_CcsC​表示第 CCC 列的整数之和，AR,CA_{R,C}AR,C​ 表示 (R,C)(R,C)(R,C) 上的整数。

容易发现，f(R,C)=rsR+csC−AR,Cf(R,C)=\mathrm{rs}_R+\mathrm{cs}_C-A_{R,C}f(R,C)=rsR​+csC​−AR,C​。

然后证明当 f(R,C)f(R,C)f(R,C) 最大时，rsR,csC≠0\mathrm{rs}_R,\mathrm{cs}_C\ne0rsR​,csC​​=0：

  • 若 rsR=csC=0\mathrm{rs}_R=\mathrm{cs}_C=0rsR​=csC​=0，则 f(r0,c0)=x0>0=f(R,C)f(r_0,c_0)=x_0>0=f(R,C)f(r0​,c0​)=x0​>0=f(R,C)，所以 (R,C)(R,C)(R,C) 不是最优解；
  • 若 rsR≠0,csC=0\mathrm{rs}_R\ne0,\mathrm{cs}_C=0rsR​​=0,csC​=0，则 f(R,c0)=rsR+csc0−AR,c0≥f(R,C)=rsRf(R,c_0)=\mathrm{rs}_R+\mathrm{cs}_{c_0}-A_{R,c_0}\ge f(R,C)=\mathrm{rs}_Rf(R,c0​)=rsR​+csc0​​−AR,c0​​≥f(R,C)=rsR​，所以 (R,C)(R,C)(R,C) 不是最优解（或有多个最优解，但其中至少有一个解 (x,y)(x,y)(x,y) 使得 rsx,csy≠0\mathrm{rs}_x,\mathrm{cs}_y\ne0rsx​,csy​​=0）
  • rsR=0,csC≠0\mathrm{rs}_R=0,\mathrm{cs}_C\ne0rsR​=0,csC​​=0 同理。

所以，我们可以依次考虑每一行 RRR（rsR≠0\mathrm{rs}_R\ne 0rsR​​=0），相当于固定了 rsR\mathrm{rs}_RrsR​。这时，我们只需找到一列 CCC（csC≠0\mathrm{cs}_C\ne 0csC​​=0），使得 csC−AR,C\mathrm{cs}_C-A_{R,C}csC​−AR,C​ 最大，就可以解决此问题。

但如果依次考虑所有包含点的列，则最坏情况下时间复杂度为 O(N2)\mathcal O(N^2)O(N2)，无法通过此题。这时，我们可以使用一个 multiset 或 map 来维护当前每列对答案的贡献（csC−AR,C\mathrm{cs}_C-A_{R,C}csC​−AR,C​）。对于每一行 RRR，仅需更新这一行上有非 000 数字的点 (R,C)(R,C)(R,C) 的贡献（减去 AR,CA_{R,C}AR,C​）即可。

这样，由于每个点会被更新正好一次，所以总时间复杂度为 O(Nlog⁡N)\mathcal O(N\log N)O(NlogN)。

代码

注意更新完成，求得当前答案后需要复原 map 或 multiset。

#include <cstdio>
#include <vector>
#include <set>
#include <unordered_map>
using namespace std;

using LL = long long;
using pii = pair<int, int>;

unordered_map<int, vector<pii>> rows;
unordered_map<int, LL> col_sum;

template <typename T>
class MaxSet {
private:
	multiset<T> s;
public:
	inline void insert(const T& x) { s.insert(x); }
	inline void update(const T& old, const T& New) {
		s.erase(s.find(old));
		s.insert(New);
	}
	inline T max() { return *s.rbegin(); }
};

int main()
{
	int n;
	scanf("%d", &n);
	while(n--)
	{
		int x, y, v;
		scanf("%d%d%d", &x, &y, &v);
		rows[x].emplace_back(y, v);
		col_sum[y] += v;
	}
	MaxSet<LL> s;
	for(auto [_, sum]: col_sum)
		s.insert(sum);
	LL ans = 0LL;
	for(auto& [x, v]: rows)
	{
		for(auto [y, val]: v)
			s.update(col_sum[y], col_sum[y] - val);
		LL cur = s.max();
		for(auto [y, val]: v)
			s.update(col_sum[y] - val, col_sum[y]), cur += val;
		if(cur > ans) ans = cur;
	}
	printf("%lld\n", ans);
	return 0;
}

G - Strawberry War

题目大意

注意本题时间限制为 6s6\mathrm s6s。

我们有一块长方形的蛋糕。它可被看作一个 H×WH\times WH×W 的网格，第 iii 行 jjj 列上有 si,js_{i,j}si,j​ 个草莓。

分析

本题解参考官方题解。

操作完成后得到的蛋糕一定是蛋糕的子矩形，所以最多只有 (H+12)×(W+12)=H(H+1)W(W+1)4≤441\binom {H+1}2\times\binom {W+1}2=\frac{H(H+1)W(W+1)}4\le 441(2H+1​)×(2W+1​)=4H(H+1)W(W+1)​≤441 种数字在剩下的块中。令这些可能的数分别为 a1,a2,…,aXa_1,a_2,\dots,a_Xa1​,a2​,…,aX​。可知 X≤H(H+1)W(W+1)4≤441X\le \frac{H(H+1)W(W+1)}4\le 441X≤4H(H+1)W(W+1)​≤441。

根据上面的 aaa，我们只需先确定 m∈{a1,a2,…,aX}m\in \{a_1,a_2,\dots,a_X\}m∈{a1​,a2​,…,aX​}，再找到与其对应的最小 MMM，算出 M−mM-mM−m 的最小值即可。

定义 fi,j,k,l,mf_{i,j,k,l,m}fi,j,k,l,m​ 表示将 x∈[i,j),y∈[k,l)x\in[i,j),y\in[k,l)x∈[i,j),y∈[k,l) 的子矩形切成 mmm 片时最小可能的每片上草莓数的最大值。

详见代码。

代码

#include <bits/stdc++.h>
using namespace std;
const long long INF = 1000000000000000000;
int main(){
  int H, W, T;
  cin >> H >> W >> T;
  vector<vector<long long>> s(H, vector<long long>(W));
  for (int i = 0; i < H; i++){
    for (int j = 0; j < W; j++){
      cin >> s[i][j];
    }
  }
  vector<vector<vector<vector<long long>>>> sum(H, vector<vector<vector<long long>>>(H + 1, vector<vector<long long>>(W, vector<long long>(W + 1, 0))));
  vector<long long> x;
  for (int i = 0; i < H; i++){
    for (int j = i + 1; j <= H; j++){
      for (int k = 0; k < W; k++){
        for (int l = k + 1; l <= W; l++){
          for (int m = i; m < j; m++){
            for (int n = k; n < l; n++){
              sum[i][j][k][l] += s[m][n];
            }
          }
          x.push_back(sum[i][j][k][l]);
        }
      }
    }
  }
  int cnt = x.size();
  long long ans = INF;
  for (int i = 0; i < cnt; i++){
    vector<vector<vector<vector<vector<long long>>>>> dp(T + 1, vector<vector<vector<vector<long long>>>>(H, vector<vector<vector<long long>>>(H + 1, vector<vector<long long>>(W, vector<long long>(W + 1, INF)))));
    for (int j = H - 1; j >= 0; j--){
      for (int k = j + 1; k <= H; k++){
        for (int l = W - 1; l >= 0; l--){
          for (int m = l + 1; m <= W; m++){
            if (sum[j][k][l][m] >= x[i]){
              dp[0][j][k][l][m] = sum[j][k][l][m];
            }
            for (int n = j + 1; n < k; n++){
              for (int o = 0; o < (n - j) * (m - l); o++){
                for (int p = 0; p < (k - n) * (m - l) && o + p < T; p++){
                  dp[o + p + 1][j][k][l][m] = min(dp[o + p + 1][j][k][l][m], max(dp[o][j][n][l][m], dp[p][n][k][l][m]));
                }
              }
            }
            for (int n = l + 1; n < m; n++){
              for (int o = 0; o < (k - j) * (n - l); o++){
                for (int p = 0; p < (k - j) * (m - n) && o + p < T; p++){
                  dp[o + p + 1][j][k][l][m] = min(dp[o + p + 1][j][k][l][m], max(dp[o][j][k][l][n], dp[p][j][k][n][m]));
                }
              }
            }
          }
        }
      }
    }
    ans = min(ans, dp[T][0][H][0][W] - x[i]);
  }
  cout << ans << endl;
}
